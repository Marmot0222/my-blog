媒体查询回答“视口有多宽”，容器查询回答“组件现在有多少空间”。当同一个组件会出现在主栏、侧栏和弹窗中时，后者更接近真实约束。

## 建立查询容器

先在组件外层声明容器，再让内部布局对容器宽度作出反应。

```css
.article-list {
  container-type: inline-size;
}

@container (min-width: 36rem) {
  .article-item {
    grid-template-columns: 12rem minmax(0, 1fr);
  }
}
```

### 处理嵌套组件

实践中要注意：

- 查询的是最近的匹配容器。
- 使用 `minmax(0, 1fr)` 避免长内容撑开网格。
- 为不支持的环境保留自然流式布局。

## 从组件约束出发

不要为每个像素增加断点。先找出排版真正失效的位置，再用少量条件切换结构。容器查询的意义是提高组件可移植性，而不是复制一套更复杂的媒体查询。
