缓存看起来只是一个键值映射，真正进入生产环境后，却会立刻遇到容量、过期、并发和恢复问题。好的实现先定义边界，再选择数据结构。

## 从最小接口开始

缓存核心接口应当足够窄，让淘汰策略和存储方式可以独立演进。

```ts
interface Cache<K, V> {
  get(key: K): V | undefined;
  set(key: K, value: V, ttl?: number): void;
  delete(key: K): boolean;
}
```

### 明确命中与过期

读取过程至少要区分三种状态：

1. 键从未存在。
2. 键存在且仍然有效。
3. 键存在但已经过期，需要清理。

这能避免用模糊的空值同时表达业务数据和缓存状态。

## 淘汰与并发控制

LRU 可以用哈希表和双向链表实现近似常数时间操作；高并发场景还需要避免同一个失效键触发大量重复回源。

- 为相同键合并进行中的加载任务。
- 为最大容量和单项大小设置硬限制。
- 持久化只记录必要数据，并允许启动时忽略损坏记录。

缓存的目标不是永不失效，而是在资源受限时提供可预测的退化方式。
