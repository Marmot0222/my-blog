## 背景与动机

AI 对话界面不只有纯文本气泡。实际产品经常需要表单、选择器、状态提示、工具结果和业务卡片共同参与一轮会话。这个 Demo 用配置化方式探索：如何在不牺牲类型约束的前提下，让服务端描述界面意图，前端负责安全地映射为已注册组件。

## 核心能力

- 以 Schema 描述消息内容与可交互 Widget。
- 使用 Widget Registry 将允许的类型映射到 Vue 组件，未知类型安全降级。
- 通过 SSE 逐步接收状态、文本和结构化数据。
- 使用 Pinia 管理会话级状态，保持协议状态与组件局部状态的边界。
- 使用 Fastify 作为轻量服务端实验入口。

## 关键技术决策

服务端不能下发任意组件名、HTML 或可执行代码。注册表是明确的信任边界：只有前端编译时注册的 Widget 才能渲染，Schema 同时约束属性形状。

流式协议按事件类型建模，而不是把所有内容拼进一个字符串。文本增量、状态变化和结构化 Widget 数据各自有清晰生命周期，前端再将它们归并进同一轮消息。

## 遇到的问题与处理

配置化很容易滑向“万能 JSON”。实验中优先限制表达能力：稳定字段进入协议，纯视觉差异留给组件，业务副作用必须通过显式事件处理器完成。这样能减少配置与实现同时失控的风险。
